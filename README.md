# Dimensionality Reduction in NLP — Practical Project

Companion to the theoretical report *"Dimensionality Reduction in NLP:
Theoretical Foundations, Methods, and Applications."* This project
implements the BoW → TF-IDF pipeline described there and benchmarks
PCA, LSA, and t-SNE on two labeled document corpora — **20 Newsgroups**
in English and **MOROCO** in Romanian — comparing how the three methods
behave across languages with different morphological complexity.

## Quick start

```bash
pip install scikit-learn numpy scipy matplotlib

# English corpus
python prepare_data.py --corpus 20news
python pca.py 20news
python lsa.py 20news
python tsne.py 20news

# Romanian corpus
python prepare_data.py --corpus moroco
python pca.py moroco
python lsa.py moroco
python tsne.py moroco
```

For 20 Newsgroups, `sklearn.datasets.fetch_20newsgroups` downloads the
corpus on first run (~14 MB, cached afterwards). If the download fails,
the pipeline transparently falls back to a synthetic newsgroup-style
corpus generated by `synthetic_corpus.py`. For MOROCO, the loader
downloads the three plain-text files directly from the official GitHub
repository (`github.com/butnaruandrei/MOROCO`) using only Python's
stdlib — no `datasets` package required. Files are cached under
`./moroco_data/` after the first run.

## Datasets

Both corpora are 6-category labeled text. The methods, evaluation
protocol, and code are identical across them; only the corpus loader
and a Romanian-specific preprocessing step (diacritic normalization,
Romanian stopwords) differ.

| | 20 Newsgroups (EN) | MOROCO (RO) |
|---|---|---|
| Source | `sklearn.datasets.fetch_20newsgroups` | `github.com/butnaruandrei/MOROCO` |
| Categories used | `comp.graphics`, `rec.sport.hockey`, `sci.electronics`, `sci.med`, `sci.space`, `talk.politics.guns` | `culture`, `finance`, `politics`, `science`, `sports`, `tech` |
| Documents | ~5,640 | ~5,400 (stratified, 900/category) |
| Vocabulary cap | 10,000 | 10,000 |
| Stopwords | sklearn English | embedded Romanian list (~270 words) |
| Special preprocessing | strip headers/footers/quotes | normalize ş→ș, ţ→ț; remove `$NE$` placeholders |

## Pipeline stages

| Stage | Script | What it does |
|------:|--------|--------------|
| 1 | `prepare_data.py --corpus <corpus>` | Load corpus, clean, build TF-IDF matrix |
| 2 | `pca.py <corpus>` | PCA + comparison vs. uncentered TruncatedSVD |
| 3 | `lsa.py <corpus>` | LSA: classification, retrieval mAP, top-terms-per-topic |
| 4 | `tsne.py <corpus>` | t-SNE perplexity sweep + linear-vs-nonlinear 2D comparison |

`<corpus>` is either `20news` or `moroco`. Each script reads from
`data/<corpus>/` and writes its results back to `data/<corpus>/` and
`figures/<corpus>/`, so the two runs never overwrite each other.

## Outputs

- **`data/<corpus>/`** — numerical artefacts: `tfidf_matrix.npz`,
  `labels.npy`, `vocabulary.json`, `dataset_stats.json`,
  `pca_results.json`, `lsa_results.json`, `tsne_results.json`, plus the
  low-dimensional embeddings as `.npy` files.
- **`figures/<corpus>/`** — every PNG produced by the experiments
  (explained variance curve, classification accuracy, LSA metrics,
  topic terms, t-SNE perplexity sweep, 2D method comparison).
- **`moroco_data/preprocessed_train/`** — cached MOROCO raw files
  (downloaded once from GitHub).

## Methods at a glance

| Method | Library call | Operates on | Reduces to |
|--------|-------------|-------------|-----------|
| PCA | `sklearn.decomposition.PCA` | Dense centered TF-IDF | k ∈ {10, 25, 50, 100, 200} |
| LSA | `sklearn.decomposition.TruncatedSVD` | Sparse uncentered TF-IDF | k ∈ {10, 25, 50, 100, 200} |
| t-SNE | `sklearn.manifold.TSNE` | 50-D LSA pre-reduction | 2D, perplexity ∈ {5, 30, 50} |

The three reduction methods are **parallel branches**, not chained
stages. The only chained step is the 50-dimensional LSA preconditioning
applied before t-SNE — standard practice (van der Maaten & Hinton 2008)
to keep t-SNE fast and noise-tolerant on text.

## Evaluation

- **PCA** — cumulative explained variance ratio + 5-fold CV
  classification accuracy with logistic regression in the reduced space.
- **LSA** — same as PCA, plus document-to-document retrieval mean
  Average Precision (mAP). Each document is treated as a query, all
  others are ranked by cosine similarity in the latent space, and AP
  is computed against the same-category set. The raw TF-IDF baseline
  is included for comparison. Top terms per latent dimension are also
  listed for qualitative topic inspection.
- **t-SNE** — silhouette score of the 2D embedding against ground-truth
  labels, plus visual comparison against PCA-2D and LSA-2D linear
  baselines.

## Headline result

LSA at small k beats raw TF-IDF for cosine-similarity retrieval on both
corpora, reproducing the classical Deerwester (1990) finding
cross-lingually:

| | EN (20 Newsgroups) | RO (MOROCO) |
|---|---|---|
| LSA mAP at k=10 | 0.595 | 0.508 |
| Raw TF-IDF mAP | 0.309 | 0.331 |
| Improvement | **+92.6%** | **+53.6%** |

The improvement is larger on English than Romanian — the opposite of
the morphological-complexity prediction. The likely reason is corpus
density: Romanian documents are longer (240 vs 179 tokens/doc) and
denser, so the raw TF-IDF baseline is already stronger, leaving less
room for LSA to improve. See section 4.2 of the report for the full
discussion.

## Requirements

- Python 3.10+
- scikit-learn ≥ 1.3
- numpy, scipy, matplotlib

No additional dependencies are needed for either corpus — MOROCO is
loaded via stdlib `urllib`, not the HuggingFace `datasets` library.
